# Utility classes — full reference

All classes below are generated by the host application's root selector when it `@include`s the build mixins from `@css-styles/common` and the parts mixins from `@css-styles/themes`. Drop them straight into HTML/Angular templates.

## Size scale

The `$sizes` map (from `@css-styles/common/_vars.scss`):

| Name | Value (px) | Alias |
|---|---|---|
| `none` | 0 | — |
| `extra-small` | 4 | `xs` |
| `small` | 10 | `sm` |
| `medium` | 15 | `md` |
| `large` | 20 | `lg` |
| `big` | 20 | `bg` |

Every spacing/border-radius/gap class has two flavours:
- bare name (e.g. `padding-small`) → **em** units (scales with parent font size)
- `-fixed` suffix (e.g. `padding-small-fixed`) → **rem** units (anchored to the root font size, ignores zoom)

Each also emits matching CSS variables: `--size-margin-small`, `--size-margin-fixed-small`, `--size-padding-small`, `--size-padding-fixed-small`, `--size-gap-small`, `--size-gap-fixed-small`, `--size-borderRadius-small`, `--size-borderRadius-fixed-small`, plus generic `--size-small` / `--size-fixed-small`.

## Margins (`buildMarginCss`)

For every `<size>` ∈ {none, extra-small, small, medium, large, big, xs, sm, md, lg, bg}:

- `margin-<size>` and `margin-<size>-fixed`
- `margin-top-<size>` / `margin-right-<size>` / `margin-bottom-<size>` / `margin-left-<size>` (+ `-fixed`)
- `margin-horizontal-<size>` (left+right) (+ `-fixed`)
- `margin-vertical-<size>` (top+bottom) (+ `-fixed`)

## Paddings (`buildPaddingCss`)

Same pattern as margins:

- `padding-<size>` / `padding-{top|right|bottom|left}-<size>` / `padding-{horizontal|vertical}-<size>` (+ `-fixed`)

## Gaps (`buildGapsCss`)

- `gap-<size>`, `column-gap-<size>`, `row-gap-<size>` (+ `-fixed`)

## Border radius (`buildBorderRadiusCss`)

- `border-round-<size>` (all four corners)
- Side pairs: `top-border-round-<size>`, `bottom-border-round-<size>`, `left-border-round-<size>`, `right-border-round-<size>`
- Single corners: `top-left-border-round-<size>`, `top-right-border-round-<size>`, `bottom-left-border-round-<size>`, `bottom-right-border-round-<size>`
- All have `-fixed` variants

> Note: in the package source the side-pair `-fixed` rules currently set `border-top-right-radius` + `border-bottom-right-radius` even for the top/bottom/left rules. This is a known quirk — prefer the non-`-fixed` side-pair variants unless you specifically need rem.

## Font sizes (`buildFontSizeCss` from `$fontSizes`)

| Class | Px (at 14px root) |
|---|---|
| `extra-extra-small-text` | 10 |
| `extra-small-text` | 12 |
| `small-text` | 14 |
| `medium-text` | 16 |
| `large-text` | 18 |
| `extra-large-text` | 20 |
| `extra-extra-large-text` | 22 |
| `extra-extra-extra-large-text` | 28 |

`-fixed` variants exist for each.

## Fixed-size CSS variables (`buildFixedSizes`)

`--size-1px` through `--size-32px`, plus matching `--size-1px-fixed` through `--size-32px-fixed`. Use these inside SCSS when you want a literal pixel value but expressed in em/rem so it scales correctly.

## Flex / display utilities (from `misc-css`)

### Display
- `block`, `inline-block`, `hidden`, `contents`, `flex`
- `print-hidden` (display:none inside `@media print`)

### Flex shape
- `flex-row`, `flex-row-reverse`, `flex-column`, `flex-wrap`, `flex-end`
- `flex-0`, `flex-025`, `flex-05`, `flex-075`, `flex-1`, `flex-2`, `flex-3`, `flex-4`, `flex-5` — each also sets `min-width: 0; min-height: 0` (essential for ellipsis-inside-flex to work)

### Grid
- `grid-subgrid`, `grid-subgrid-columns`, `grid-subgrid-rows`
- `grid-whole-row` (`grid-column: 1 / -1`)
- `grid-first-cell` (`grid-column: 1/2; grid-row: 1/2`)

### Alignment
- `align-items-{center|start|end}`
- `align-self-{start|center|end|stretch}`
- `align-content-{start|center|end}`
- `justify-self-{start|center|end|stretch}`
- `justify-content-{start|center|end|stretch|space-between|space-around|space-evenly}`

### Positioning
- `relative`, `absolute`, `left-0`, `top-0`, `right-0`, `bottom-0`
- `va-middle`, `va-top` (vertical-align for inline contexts)

### Sizing
- `full-width`, `half-width`, `third-width`, `quarter-width`, `full-height`, `max-height-full`
- `min-width-none`, `min-height-none`, `min-dimensions-none`, `max-width-none`

### Overflow / scroll
- `overflow-hidden`, `overflow-auto`
- `overflow-vertical-auto`, `overflow-vertical-scroll`
- `overflow-horizontal-auto`, `overflow-horizontal-scroll`
- `thin-scrollbar`, `table-container`

### Text
- `text-{left|center|right}`
- `text-{uppercase|capitalize|small-caps}`, `first-letter-uppercase`
- `bold`, `semi-bold` (500), `regular` (400), `semi-thin` (300), `ultra-thin` (200), `italic`
- `text-ellipsis` (overflow:hidden + text-overflow:ellipsis + white-space:nowrap)
- `flexible-ellipsis` (the `min-width: 0; min-height: 0` half — use on the parent flex child if it's not already `flex-N`)
- `white-space-{normal|nowrap|pre|pre-wrap}` and aliases `content-wrap` / `content-nowrap`
- `spread-text` (letter-spacing: 1px)
- `line-height-1em`, `font-1em`
- `text-selection` (user-select: text)

### Cursors & misc
- `cursor-pointer` / `pointer-cursor`
- `cursor-not-allowed` / `not-allowed-cursor`
- `notransition`, `transition-all-400`
- `reset-scale` / `reset-zoom` / `root-scale` (restore `font-size: 1rem` inside a `fluid-type` container)

## %placeholders (for component SCSS)

Every class above has a matching `%placeholder` you can `@extend` from your component's SCSS (`@use '@css-styles/common'` first). For example: `%flex-row`, `%flex-1`, `%align-items-center`, `%text-ellipsis`, `%bold`, `%overflow-auto`, `%full-width`. Use them when many declarations would otherwise repeat; for one-off styles, just write the property.

## Component classes (from themes parts)

### Buttons (`css-buttons`)
- `btn-<variant>` — built-in variants: `primary`, `secondary`, `success`, `info`, `warning`, `danger`, `error`, `default`, plus any custom variants the host app passes via `$buttonTypes` (`link`, `my-new`, …)
- Base styling (padding, font, border, transition, all driven by `--button-*`) is applied via the attribute selector `[class|="btn"]` since v2.6.0, so the **single class form is sufficient**: `<button class="btn-primary">`. The legacy two-class form `<button class="btn btn-primary">` still applies (the `.btn` selector remains for backwards compatibility).
- Since v2.6.0, each variant's `background` / `foreground` defaults to `var(--theme-<variant>)` / `var(--theme-on<Variant>)`. To add a new variant it's often enough to register it via `$buttonTypes` and add `theme.<variant>` + `theme.on<Variant>` colors — no per-variant button map needed.
- States: `:hover`, `:focus`, `:active`, `:disabled` / `.disabled` — automatic
- Modifiers:
  - `.icon-only` — strips padding (used for icon-only buttons)
  - `.only-content` — text/icon-only variant (transparent background); each variant has dedicated CSS vars `--button-<variant>-onlyContent-*`
- Group container: `.buttons-gap` — applies `--buttons-columnGap` / `--buttons-rowGap`

### Forms (`css-forms`)
- `form-group` — flex container for a label/input pair (direction from `--formGroup-direction`, default column)
- `form-control` — input/select/textarea base (border, padding, focus, disabled)
- `form-control-static` — read-only text styled like an input
- `input-group` + `input-group-addon` — prefix/suffix elements
- `control-label` — label above an input
- `has-error` — wrapper that turns child inputs/labels red and shows validation styling
- `.validation-error-div > div` — error message styling
- `.form-error` — list-item error entry
- `inputs-gap` — `column-gap`/`row-gap` from `--inputs-*` for rows/columns of fields

### Alerts (`css-alerts`)
- `alert-<type>` — `info`, `warning`, `success`, `danger`, `error` (configurable)
- Base padding/border-radius/margin are applied via the attribute selector `[class|="alert"]` since v2.6.0, so the single class is sufficient: `<div class="alert-warning">`. The legacy two-class form `alert alert-warning` still works.

### Blocks (`css-blocks`)
- `<name>-block` — themed panel. Default config exposes `highlight-block`. Host apps may enable additional ones (e.g. `primary-block`, `secondary-block`) by passing `$blockTypes` to `css-blocks` and adding matching keys to the theme map.

### Titles (`css-titles`)
- `page-title` — big themed header bar at the top of a page
- `section-title` — uppercase divider
- `subsection-title` — smaller uppercase divider

### Texts (`css-texts`)
- `text-<type>` — `primary`, `danger`, `warning`, `success`, `info` (background + foreground from theme)
- `text-<type>-transparent` — foreground only

### Common (`css-common`)
- `<hr>` is styled (top border in `--misc-separator-foreground`)
- `semi-tight` (font-size .9em), `tight` (font-size .8em)
- `text-selection`, `text-selection-disabled`
- `line-height-default` (1em)

### Animations (`provideAllAnimations`)
- `slide-in`, `slide-out`, `slide-in-animation`
- `fly-in`, `fly-out`
- `fade-in`, `fade-out`

Each animation reads `--<name>-duration` and `--<name>-to-*` / `--<name>-from-*` CSS variables so you can tune per scope. Defaults are 400ms ease-in/out.
